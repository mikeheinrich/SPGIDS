Components

Reusable UI components built with our design system. Each component is self-contained and can be easily integrated into your projects.

Buttons

Button components provide consistent call-to-action elements. Based on Adobe XD specifications.

Primary Button

The primary button is used for the main call-to-action. Specifications:

  • Dimensions: Auto width (min 90px) × 32px height
  • States: Default, Hover, Disabled, Disabled Hover
  • Hover Transition: Auto-Animate, Ease Out, 0.3s
  • Disabled Hover: Dissolve animation, 0.3s
  • Colors: Filled background using primary colors

Primary Button States

Default: Primary blue background (#006d89)
Hover: Darker blue (#00495b) with smooth transition
Disabled: Gray background with reduced opacity on hover

<!-- Include CSS --> <link rel="stylesheet" href="components/button-component/button-component.css"> <!-- Primary Button --> <button class="btn-primary">Button</button> <!-- Disabled State --> <button class="btn-primary" disabled>Disabled</button> <!-- Size Variations --> <button class="btn-primary btn-small">Small</button> <button class="btn-primary">Default</button> <button class="btn-primary btn-large">Large</button>

Secondary Button

The secondary button is used for secondary actions. Specifications:

  • Dimensions: Auto width (min 87px) × 32px height
  • States: Default, Hover, Disabled, Disabled Hover
  • Hover Transition: Auto-Animate, Ease Out, 0.3s
  • Disabled Hover: Dissolve animation, 0.3s
  • Style: Outlined with 1px border using primary color

Secondary Button States

Default: Transparent background with primary blue border (#006d89)
Hover: Light gray background (#eeeeee) with primary blue text
Disabled: Gray border and text with reduced opacity on hover

<!-- Secondary Button --> <button class="btn-secondary">Button</button> <!-- Disabled State --> <button class="btn-secondary" disabled>Disabled</button> <!-- Size Variations --> <button class="btn-secondary btn-small">Small</button> <button class="btn-secondary">Default</button> <button class="btn-secondary btn-large">Large</button>

Button Group

Button groups act like tabs or radio buttons, allowing only one selection at a time. The active button has a gray background, red text, and a red top border.

Button Group States

Button groups provide a tab-like interface where buttons are visually connected. Specifications:

  • Dimensions: 90px × 32px per button
  • States: Active, Inactive, Hover, Disabled
  • Active: Gray background (#eeeeee), red text (#d6002a), red top border (2px)
  • Inactive: Transparent background, gray text (#5c5c5c), gray borders
  • Behavior: Radio button-like - only one active at a time

Accordion

Accordion components allow content to be expanded and collapsed. The header shows a chevron icon that rotates when opened.

Accordion States

Accordions provide expandable/collapsible content sections. Specifications:

  • Dimensions: Flexible width × 32px height (closed)
  • States: Default (closed), Hover, Open, Disabled
  • Default: White background, dark gray text (#1a1a1a), gray top border (#5c5c5c)
  • Hover: Bold text (#32363b), red top border (#d6002a), subtle shadow
  • Open: Bold text, red top border, shadow, chevron rotated down
  • Icon: Red chevron, rotates from right (closed) to down (open)

Closed State

This is the accordion content. It can contain any HTML content including text, images, lists, and other components.

Open State

This accordion is open by default. The chevron icon points down when open. You can click the header to collapse it.

Default: White background, dark gray text (#1a1a1a), gray top border
Hover: Bold text (#32363b), red top border (#d6002a), subtle shadow
Open: Bold text, red top border, shadow, chevron rotated 90deg
Animation: Smooth height transition (0.3s ease)

Alert

Alert components display messages with different states. They can be dismissed with a close button.

Alert Variants

Alerts provide a flexible way to display messages to users. Specifications:

  • Dimensions: Flexible width × 40px height (min)
  • Variants: Default, Success, Attention
  • Default: Light gray background (#eeeeee), gray top border (#5c5c5c), no leading icon
  • Success: Light green background, green border, green checkmark icon
  • Attention: Light orange background, orange border, orange warning icon
  • Close Button: Dark gray circle with white X on the right

List Group

A simple, flat list of selectable items. Based on the left navigation component but simplified to remove hierarchy.

List Group States

List groups provide a simple interface for selecting from a list of items. Specifications:

  • Dimensions: 361px width (default, flexible) × 32px height per item
  • States: Default, Hover, Active, Disabled
  • Default: Transparent background, dark gray text (#32363b)
  • Hover: Subtle background tint (rgba(0, 0, 0, 0.02))
  • Active: Light gray background (#f0f0f0), red text (#d6002a), red top accent bar (3px)
  • Dividers: Optional 1px gray dividers between items

Basic List Group

    Hover: Subtle background tint
    Active: Light gray background (#f0f0f0), red text (#d6002a), red top accent bar
    Keyboard: Enter/Space to select, arrow keys for navigation

    In-Page Nav

    Hierarchical navigation component with parent and child structure. Supports two-level navigation with visual hierarchy.

    In-Page Nav Structure

    In-page navigation provides hierarchical navigation with parent and child levels. Specifications:

    • Dimensions: 361px width × flexible height
    • Structure: Parent items (bold, 10px padding) with nested Child items (normal, 20px padding)
    • States: Default, Hover, Active
    • Parent Items: Bold font (700), 10px left padding, dark gray text (#32363b)
    • Child Items: Normal font (400), 20px left padding, dark gray text (#32363b)
    • Active: Light gray background (#f0f0f0), red text (#d6002a), red top accent bar (3px)
    • Dividers: 1px gray dividers between all items

      Parent Items: Bold, 10px padding, represent top-level entities
      Child Items: Normal weight, 20px padding, nested under parents
      Active: Light gray background, red text, red top accent bar
      Hover: Subtle background tint on all items

      Alert

      Banner-style alerts for displaying important messages to users. Supports multiple variants and can be dismissed.

      Alert Variants

      Alerts provide contextual feedback messages. Specifications:

      • Variants: Success, Warning, Error, Info
      • Features: Icon, title (optional), message, dismissible close button
      • Colors: Variant-specific background and border colors
      • Layout: Flex layout with icon, content, and close button

      Alert Examples
