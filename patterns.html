Patterns

Patterns are groups of components that work together to create functional parts of an interface. These patterns provide reusable solutions for common UI challenges.

About Patterns

Patterns combine multiple components to solve specific user interface problems. Each pattern is designed to be reusable and adaptable to different contexts while maintaining consistency across the design system.

File Upload

A drag-and-drop file upload pattern with file list display. Supports both file browsing and drag-and-drop, with the ability to display uploaded files either below or to the right of the drop zone.

Vertical Layout

The file list appears below the drop zone. Best for narrow containers or when the list can grow long.

Horizontal Layout

The file list appears to the right of the drop zone. Best for wider containers where horizontal space is available.

Usage

<!-- Include CSS --> <link rel="stylesheet" href="patterns/file-upload-pattern/vanilla/file-upload-pattern.css"> <!-- Include JavaScript --> <script src="patterns/file-upload-pattern/vanilla/file-upload-pattern.js"></script> <!-- Container --> <div id="file-upload-container"></div> <!-- Initialize --> <script> initFileUpload({ containerId: 'file-upload-container', layout: 'vertical', // or 'horizontal' dropzoneText: 'Drag and drop file to upload', acceptedFormats: 'Accepted formats: pdf, doc, docx', accept: '.pdf,.doc,.docx', multiple: true, maxFileSize: 10 * 1024 * 1024, // 10MB maxFiles: 10, browseButtonText: 'Browse files', fileListTitle: 'Uploaded files', onFilesChange: function(files) { console.log('Files:', files); } }); </script>

Query Builder

A 3-column pattern for building queries by selecting attributes across multiple categories. The first column displays categories, the second column shows selectable attributes with search, and the third column displays a summary of selections grouped by category.

Query Builder Pattern

The Query Builder pattern provides a comprehensive interface for selecting attributes across multiple categories. Specifications:

  • Layout: 3-column layout with fixed widths
  • Column 1: Category list (256px) using list-group component
  • Column 2: Attributes with search and checkboxes (flexible width)
  • Column 3: Summary of selections grouped by category (320px)
  • Height: 600px with scrollable content areas
  • Features: Search filtering, checkbox selection, tag removal, save/cancel actions

Query Builder Example

Categories: Click on a category in the left column to view its attributes
Search: Use the search box to filter attributes in the current category
Selection: Click checkboxes to move attributes from center to right column
Summary: View all selected attributes grouped by category, click X to remove items and move them back to center

Right Toolbar

A flexible vertical toolbar pattern positioned on the right side of the screen. Supports single actions, actions that open panels (flyouts), and actions that open modals.

Right Toolbar Pattern

The Right Toolbar provides a consistent interface for tool actions. Each item has an icon and text label, and can trigger single actions, open flyout panels, or open modal dialogs.

  • Position: Fixed to the right side of the screen (50px wide)
  • Actions: Single click handlers, panel flyouts, or modal dialogs
  • Icons: Uses design system icons with teal color (#006d89)
  • Panels: Appear to the left of the toolbar (320px wide)
  • Modals: Centered on screen using modal component
